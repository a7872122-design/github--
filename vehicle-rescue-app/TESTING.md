## 🧪 完整端對端測試流程

### 場景 1: 用戶請求救援 → 廠商接單

#### 步驟 1: 用戶應用 (User Side)
1. 打開 **http://localhost:5173**
2. 等待位置授權提示（允許或使用模擬座標）
3. 應看到：
   - ✅ 地圖加載並顯示用戶位置
   - ✅ 5家維修廠列表顯示在右侧
   - ✅ 每家廠顯示距離、評分、電話、技能

#### 步驟 2: 觸發救援請求
1. 在廠商卡片上點擊 **"◀滑動請求救援▶"** 按鈕
2. **預期結果**：
   - ✅ 按鈕變為 "請求中..." 狀態
   - ✅ 右下角出現藍色 "✅ 救援請求已發送" 通知
   - ✅ 3秒後通知自動消失
   - ✅ 後端伺服器 console 顯示：`🆘 救援請求: REQ-{timestamp} -> 廠商 {廠商ID}`

#### 步驟 3: 廠商接單 (Admin Side)
1. 打開新標籤 **http://localhost:5173?admin=true**
2. 登錄表單出現，輸入：
   - **廠商 ID**: 選擇對應的廠商 (例: 1, 2, 3...)
   - **廠商名稱**: 例「24小時快速維修廠」
   - 點擊 **"登入系統"**

3. **預期結果**：
   - ✅ 廠商管理後台加載
   - ✅ 後端 WebSocket 連接確認信息
   - ✅ 左側「救援請求」佇列更新
   - ✅ 黃色 "待接單" 徽章顯示新請求
   - ✅ 頂部顯示廠商名稱和在線狀態

#### 步驟 4: 廠商接受救援
1. 在左側請求佇列中點擊新請求
2. 點擊右側 **"✓ 接受救援"** 按鈕
3. **預期結果**：
   - ✅ 請求卡片變為綠色 "已接單" 狀態
   - ✅ 右下角橙色通知：`✅ 已接受 REQ-{requestId}`
   - ✅ 地圖顯示用戶位置和廠商位置
   - ✅ 底部按鈕變為 **"🚗 出發"** 

#### 步驟 5: 廠商狀態更新
1. 廠商板上點擊 **"🚗 出發"** 按鈕
2. **預期結果**：
   - ✅ 用戶應用右下角出現黃色通知：`🚗 {廠商名稱} 正在趕往...`
   - ✅ 廠商請求狀態變為藍色 "出發中"
   - ✅ 按鈕變為 **"✨ 已到達"**

3. 廠商板上點擊 **"✨ 已到達"** 按鈕
4. **預期結果**：
   - ✅ 用戶應用右下角出現紫色通知：`✨ {廠商名稱} 已到達現場`
   - ✅ 廠商請求狀態變為紫色 "已到達"
   - ✅ 按鈕變為 **"✓ 完成"**

#### 步驟 6: 完成救援
1. 廠商板上點擊 **"✓ 完成"** 按鈕
2. **預期結果**：
   - ✅ 請求從佇列中移除
   - ✅ 廠商板顯示 "暫無救援請求"

## 🛠️ 常見問題

### Q: 廠商沒收到請求通知？
**A**: 檢查：
- [ ] 廠商 ID 是否與用戶請求的廠商 ID 匹配 (必須相同)
- [ ] 廠商是否已完成登錄和Web Socket 連接
- [ ] 後端伺服器 console 中廠商是否註冊成功
- [ ] 防火牆是否擋住 WebSocket (3001 端口)

### Q: 地圖不顯示？
**A**: 檢查：
- [ ] Google Maps API 金鑰是否在 `.env.local` 中
- [ ] API 金鑰是否有效
- [ ] 瀏覽器 console 中是否有 CORS 錯誤

### Q: WebSocket 連接失敗？
**A**: 檢查：
- [ ] 後端伺服器是否在 3001 端口運行
- [ ] 是否有其他進程佔用 3001 端口
- [ ] 防火牆設置

```bash
# 檢查 3001 端口
Get-NetTCPConnection -LocalPort 3001
```
